# cova 사용 매뉴얼 (VitePress) — 설계 문서

> cova 스튜디오 · 뷰어 · Figma 플러그인 3개 제품의 사용 매뉴얼
> 작성일: 2026-06-24

## 1. 개요

cova의 세 제품에 대한 **사용자용 매뉴얼**을 VitePress로 만든다.
제품별 독자가 다르므로(편집자·관리자 / 열람자·게스트 / 디자이너) 상단 네비로
세 제품을 분리하고, 각 제품은 메뉴 단위로 페이지를 둔다.

- 언어: **한국어**
- 위치: 저장소 루트의 **`manual/`** (설계문서용 `docs/`와 분리)
- 범위: 모든 메뉴의 **본문 전체 작성**(실제 동작을 코드/UI 기준으로 확인해서 기술)
- 이미지: **사용자가 직접 준비** → 본문에는 이미지 자리만 플레이스홀더로 남긴다.

## 2. 기술 구성

`manual/`을 **독립 서브패키지**로 둔다(`figma-plugin/`과 동일 패턴: 자체
`package.json` + `node_modules`). 메인 Next.js 앱의 빌드/의존성과 섞이지 않는다.

| 항목 | 선택 |
|------|------|
| 도구 | VitePress 최신 1.x |
| 언어 설정 | `lang: 'ko-KR'` |
| 검색 | VitePress 기본 로컬 검색(`search.provider: 'local'`) |
| 실행 | `manual/` 안에서 `npm run docs:dev` / `docs:build` / `docs:preview` |
| 배포 | 이번 범위 아님(정적 빌드 산출물만 생성 가능) |

- 메인 앱 루트 `package.json`은 건드리지 않는다(서브패키지로 자족).
- `manual/.gitignore`에 `node_modules`, `.vitepress/cache`, `.vitepress/dist` 포함.

## 3. 정보구조(IA)

### 3.1 상단 네비

```
홈 | 스튜디오 | 뷰어 | Figma 플러그인
```

### 3.2 사이드바 / 페이지 구성 (메뉴 1개 = 페이지 1개)

```
manual/
├─ index.md                      홈 — 3개 제품 소개 + 바로가기 카드(VitePress home layout)
│
├─ studio/   (편집자·관리자)
│  ├─ index.md                   스튜디오 소개 + 시작하기(가입·로그인·승인 대기 흐름)
│  ├─ proposals.md               시안 — 목록 보기, 새 시안 만들기
│  ├─ proposal-detail.md         시안 상세 — 버전 관리, 공개/비밀번호 설정, 태그 부여
│  ├─ tags.md                    태그 설정 (admin 전용)
│  ├─ users.md                   사용자 관리 — 가입 승인·역할 변경 (admin 전용)
│  └─ account.md                 내 계정 (/me)
│
├─ viewer/   (열람자·게스트)
│  ├─ index.md                   뷰어 소개 + 공개 링크 열기·비밀번호 입력
│  ├─ preview.md                 프리뷰 — 풀스크린 슬라이드 / 캔버스 뷰 전환
│  ├─ collaboration.md           실시간 회의 — 커서·핀 코멘트·채팅
│  └─ guest.md                   게스트 이름 (코멘트/채팅 첫 작성 시 입력)
│
└─ plugin/   (디자이너)
   ├─ index.md                   플러그인 소개 + 설치(매니페스트 임포트)
   ├─ login.md                   로그인 — 외부 브라우저 페어링
   ├─ new-proposal.md            새 시안 만들기 (프레임 선택)
   └─ new-version.md             기존 시안에 새 버전·새 안(A·B) 올리기
```

총 16개 페이지.

### 3.3 사이드바 동작

- 경로 기반 멀티 사이드바: `/studio/`, `/viewer/`, `/plugin/` 각각 별도 사이드바.
- 각 사이드바 최상단에 해당 제품 개요(`index.md`) 링크.

## 4. 콘텐츠 작성 규칙

각 페이지는 다음 골격을 따른다(분량은 메뉴 복잡도에 맞게 조절):

1. **무엇을 하는 화면인가** (1~2문장)
2. **누가 쓸 수 있나 / 사전 조건** (권한·로그인·승인 등)
3. **단계별 사용법** (번호 목록)
4. **참고/주의** (제약, 흔한 실수 — 필요 시)

- 화면 명칭·버튼 라벨은 **실제 UI 문구**(코드 기준)를 그대로 쓴다.
  예: 사이드바 "시안 / 태그 설정 / 사용자 관리", "홈으로".
- 권한 구분이 있는 메뉴(태그·사용자 관리=admin)는 **관리자 전용**임을 명시.
- 제품 간 연결 동작은 상호 링크(예: 플러그인 업로드 → 스튜디오 시안 상세).

### 4.1 이미지 플레이스홀더 규칙

이미지가 필요한 자리는 **VitePress 커스텀 컨테이너**로 빈 자리를 남긴다.
사용자가 나중에 캡처를 끼워넣을 위치와 내용을 한 줄로 설명한다.

```md
::: info 🖼 이미지 자리
스튜디오 시안 목록 — 좌측 사이드바 + 카드 그리드가 보이는 전체 화면
:::
```

- 실제 `<img>`/마크다운 이미지는 넣지 않는다(파일이 없으므로 깨짐 방지).
- 한 페이지에 필요한 만큼 여러 개 둘 수 있다.

## 5. 산출물 / 검증

- `manual/` 서브패키지 + `.vitepress/config.mts` + 16개 `.md` 페이지.
- `npm install && npm run docs:build`가 **무오류로 통과**(깨진 링크 없음)하는 것으로 검증.
- 본문 내용의 사실성은 각 페이지 작성 시 해당 기능의 코드/라우트를 직접 확인해 보장.

## 6. 범위 밖 (YAGNI)

- 다국어(i18n) — 한국어만.
- 실제 스크린샷/이미지 파일 — 사용자가 준비.
- 배포 파이프라인(호스팅) — 정적 빌드까지만.
- 메인 앱 루트 `package.json` 스크립트 추가 — 서브패키지로 자족.
