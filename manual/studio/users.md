# 사용자 관리

**사용자 관리**(`/studio/users`)에서는 가입한 사용자의 **권한(역할)** 을 관리합니다.
새로 가입한 사람을 승인하거나, 역할을 바꿉니다.

::: warning 관리자 전용
이 메뉴는 **관리자(admin)** 에게만 보입니다.
:::

## 역할 종류

| 역할 | 설명 |
|------|------|
| **대기**(pending) | 가입 직후 상태. 아무 권한이 없어 [승인 대기](/studio/) 화면만 보입니다. |
| **편집자**(editor) | 승인된 편집자. **모든 시안**을 보고 만들고 수정·삭제할 수 있습니다. |
| **관리자**(admin) | 편집자 권한 + 가입 승인·역할 변경 + 태그 설정 가능. |

## 사용자 목록

상단에 제목 **"사용자 관리"** 와 검색창(*"이름·이메일 검색"*)이 있습니다. 검색 결과 개수는
**"전체 N개"** 로 표시됩니다.

표 컬럼은 다음과 같습니다.

- **이름** — 표시 이름(없으면 `—`)
- **이메일**
- **가입수단** — 이메일/구글 등 가입 방법
- **가입일**
- **작업** — 역할 선택 드롭다운

::: info 🖼 이미지 자리
사용자 관리 — 검색창과 사용자 표(이름·이메일·가입수단·가입일·역할 드롭다운)
:::

## 승인하기 / 역할 바꾸기

각 사용자 행의 **"작업"** 드롭다운에서 역할을 선택합니다.

- **대기** → 아직 권한 없음(가입 직후 기본값)
- **편집자** → 시안 관리 권한 부여(= 승인)
- **관리자** → 관리 권한까지 부여

선택하면 즉시 적용되고 *"역할을 변경했습니다"* 안내가 뜹니다. 새로 가입한 사람을
**편집자** 또는 **관리자** 로 바꾸는 것이 곧 "가입 승인"입니다.

::: tip 관리자는 여러 명 둘 수 있어요
관리자는 다른 사용자를 관리자로 승격할 수 있습니다. 운영 인원에 맞게 여러 명을 두세요.
:::
